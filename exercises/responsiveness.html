Responsiveness

As we build our websites, we are most likely writing them on a single computer and using a specific monitor or browser size to test them. But, as we know, there are many possible screen sizes that exist and we can’t assume that our website visitors will be using the exact same screen size as us. So, we will want to try to build our websites responsively, so that they can adapt to the variety of device demensions and scales as best as possible.

In order to prepare our pages to be responsive we should include this meta tag in the <head> element of every html document:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

This will let the browser know how it should adjust the dimensions and scaling of elements on the page depending on the device being used.

However, this is just the first step. This will account for making most of the default behaviors and scaling of html elements responsive, but there’s a good chance that our website structures and layouts will be more complex than just unstyled html documents. To exercise more control over our pages’ responsiveness, we can use media queries to define browser-size breakpoints.
